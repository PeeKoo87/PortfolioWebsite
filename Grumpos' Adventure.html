Grumpo's Adventure

Introduction


The game is my first actual game I have ever made,
Grumpo's Adventure is my first year school portfolio project assignment.
I started with only the basic knowledge of c# and unity at the start of the project and had to learn as the project progressed.
While the game is still in demo phase it has been a wonderful experience, although quite challenging considering the timeframe.

In the sections below I will go in to detail on the game itself, skills and software I needed to learn for making the game.

Graphics

Graphics were partly made using photoshop and most of the assets are from unity asset store,
player character and the goblin enemy both have bone structures,
and were animated through manipulating the bone structures within unity animator.
Backgrounds have a parallax effect implemented on them to make a smoother feel while playing the game,
backgrounds and level graphics have been imported from unity asset store using free assets for this project.

Audio

Sounds for the game were implemented using FMOD, i started learning FMOD usage for the project and still has room for improvement, although i did get a grasp of the basics.
I Implemented the sounds for the player and enemy within the animator for more precise timing,
i did fiddle around with implementing sound effects through code,
but since this was my first solo project using animator was within my range of skills and easiest to implement within the timeframe of the project.

Animation

Animations within the game function based on a statemachine, I found it the most efficient way of doing things regarding this game.
The animation logic is separated within base states and substates to keep the process more organized,
player animations are divided to two main states ground- and air states which separate things like air attack animations,
falling, basic movement and attack patterns.

Save And UI

Save and UI functionality was the most challenging part of the project, the save system in game uses a data persistence system,
which saves all collected item, player position and encrypts save game data.
The game data is saved in a json file which encrypted and decrypted during the save and load functions,
items have their own unique randomly generated id:s to make sure there are no duplicate values within the serialized dictionaries within the script.
UI functions are operated through code as one would expect and were fairly straightforward to implement,
Menu option are separated to their own respective scripts to keep the functions organized and easier to read.
In regard of the data persistence scripts i have to thank Shaped By Rain Studios on youtube for very easy to understand tutorials,
they were very educational and easy to follow despite the very complex subject of data persistence.

Reflection

Grumpo's Adventure has be quite the adventure as a project, i had to learn most skills from the very beginning.
FMOD was a completely unfamiliar software to work with, although while making this game i learnt the very basics on how to operate it,
a good skill to have in my opinion, even if only on the very basic level.
The process of making Grumpo's adventure also deepend my undestanding of unity and c# programming,
although it has been an intense process it has been a very exciting project to work on!

On hindsight Fmod might have been a slight overkill on the project since most sound effect could have been implemented
in a more simple fashion through animator and code itself.
Slight fumbling on the order of things to be made happened aswell but, i did learn my lesson, better planning makes for a smoother process!.

The game is still a work in progress, but I intend to polish the game and my skills in the process.
All in all if has been a very fruitful project in regard of gaining perspective and insight on game development process as a whole.
